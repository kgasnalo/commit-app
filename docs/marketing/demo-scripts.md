## Video Specifications

### Twitter Video Requirements
- **Format**: MP4 (H.264 codec, AAC audio)
- **Resolution**: 1080 x 1920 (9:16 portrait) or 1920 x 1080 (16:9 landscape)
- **Frame Rate**: 30fps (minimum), 60fps (recommended)
- **Duration**: 15-140 seconds (optimal: 15-30 seconds for engagement)
- **File Size**: Max 512MB (but aim for <10MB for faster loading)
- **Aspect Ratio**: 9:16 (portrait, mobile-first) or 1:1 (square, universal)

### Recommended Settings
- **Portrait (Mobile App Demos)**: 1080 x 1920, 60fps, 15-30 seconds
- **Square (Universal)**: 1080 x 1080, 60fps, 15-30 seconds
- **Landscape (Infographics/Comparisons)**: 1920 x 1080, 30fps, 15-45 seconds

## Production Workflow

### Phase 1: Pre-Production (1 day)
- [ ] Finalize scripts (all 3 main videos + 5 snippets)
- [ ] Prepare seed data (see `screenshot-guide.md`)
- [ ] Set up screen recording tools
- [ ] Download royalty-free music (Epidemic Sound, Artlist, or YouTube Audio Library)
- [ ] Create Figma mockups for Video 3

### Phase 2: Filming (1-2 days)
- [ ] Screen record all app interactions (iOS Simulator or device)
- [ ] Capture multiple takes (3-5 per shot)
- [ ] Record voiceover (if using)
- [ ] Export raw footage

### Phase 3: Editing (2-3 days)
- [ ] Import footage into editing software (iMovie, Final Cut Pro, or CapCut)
- [ ] Cut and arrange shots per script
- [ ] Add text overlays (bold, readable fonts)
- [ ] Add transitions (fade, swipe, zoom)
- [ ] Add music (mix to 30-40% volume for background)
- [ ] Color grading (enhance Titan palette)
- [ ] Export drafts for review

### Phase 4: Review & Revise (1 day)
- [ ] Watch all videos start to finish
- [ ] Check for:
  - Typos in text overlays
  - Timing issues (too fast/slow)
  - Audio sync (if voiceover)
  - Readability (especially on mobile)
- [ ] Make revisions
- [ ] Export final versions

### Phase 5: Export & Upload (1 day)
- [ ] Export all videos per specs above
- [ ] Compress if file size > 10MB (use HandBrake)
- [ ] Upload to Twitter as drafts
- [ ] Test autoplay (silent mode)
- [ ] Schedule posts per content calendar

**Total Timeline**: 5-7 days (can be done in parallel with other marketing tasks)

## Video Posting Strategy

### X (Twitter) Video Best Practices
1. **Autoplay**: Videos autoplay muted. Use text overlays, not voiceover-dependent content.
2. **First 3 Seconds**: Hook viewers immediately (show problem or result first).
3. **Captions**: Always add captions (Twitter's accessibility feature).
4. **Aspect Ratio**: Portrait (9:16) performs best on mobile.
5. **Length**: 15-30 seconds optimal (longer videos have lower completion rates).

### Posting Schedule
- **Video 1 (30s Pitch)**: Day 33 (Friday, Week 5) - Demo video post
- **Video 2 (Monk Mode)**: Day 13 (Saturday, #screenshotsaturday) - Can be video instead of image

### Ken Burns Effect
- **What**: Slow zoom/pan on static images
- **Use**: Book pile shot, logo end card

### Zoom Punch
- **What**: Quick zoom in on UI element

- **How**: Keyframe scale (1.0 â†’ 1.2) over 0.5 seconds

### Parallax
- **What**: Background moves slower than foreground
- **Use**: Multi-layer UI (cards over background)
- **How**: Motion tracking in After Effects or Screen Studio

### Glow Pulse
- **What**: Orange glow pulsates on key elements

- **How**: Keyframe opacity on glow layer (0.3 â†’ 0.6 â†’ 0.3)
