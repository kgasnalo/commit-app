# COMMIT X - Pre-Launch Marketing Strategy v2

## Executive Summary
X Communities（"Build in Public" 180K+メンバー）を軸に、フォロワー0からでもリーチを確保する戦略。ミックス型トーン（強いフック + 静かな本文）と5-7投稿/日の高頻度で、8週間で3,000フォロワーを目指す。

**Timeline**: 8 weeks
**Budget**: $0-50
**Created**: 2026-01-20
**Last Updated**: 2026-01-23（v2 全面改訂）

## X Community戦略（最重要）

### なぜCommunityか
- フォロワー0でも180K人にリーチ可能
- アルゴリズムがCommunity投稿を優遇
- 事例: 30日で2,000フォロワー獲得（Community活用者）

### ターゲットCommunity
1. **"Build in Public"**（180K+メンバー）← メイン
2. **"Indie Hackers"**（50K+メンバー）
3. **"Startup"**（100K+メンバー）

### Communityルール
- 3,000フォロワーに達するまでは**100%**メイン投稿をCommunityに投稿
- マイクロ投稿もCommunityに（リーチ確保）
- 3,000フォロワー以降はタイムライン投稿に移行

## トーン：ミックス型

### 構造
```
[1行目] フック — 強い。数字。意外性。スクロールを止める。（Marc Lou的）
[2行目〜] 本文 — 事実ベース。短い文。説明しすぎない。（自分のトーン）
```

### フック（1行目）のルール

- **110文字以下**
- 数字 or 意外性を含む
- 4パターンから選ぶ:

| パターン | 例（英語） | 例（日本語） |
|---------|-----------|------------|
| Bold Statement | "Reading apps don't solve the reading problem." | 「読書アプリは、読書の問題を解決しない。」 |
| Number Hook | "4 apps. 3 years. 0 books finished." | 「4つのアプリ、3年、0冊読了。」 |
| Pattern Interrupt | "I stopped buying books. I bet ¥2,000 instead." | 「本を買うのをやめた。代わりに¥2,000を賭けた。」 |
| Contrast | "72% hit their goals. The secret: pricing failure." | 「72%が目標達成。秘密は、失敗に値段をつけたこと。」 |

### 本文のルール
- 1文は短く
- 感情を直接説明しない
- 広告っぽい言い切り禁止
- 事実を淡々と述べる
- フック直後は「落差」を作る（強い1行→静かな本文）

### Before/After（トーン比較）

**Before（静かすぎ）:**
```
本棚には23冊の積読本がある。
「いつか読む」と言い続けて、
そのまま何年も経っていた。
```

**After（ミックス型）:**
```
23冊。3年。0冊読了。

「いつか読む」と言い続けて、
そのまま何年も経っていた。

だから、逃げ道を消すアプリを作った。
```

## エンゲージメント戦略

### デイリールーティン（45分/日）

| 時間 | アクション | 所要時間 |
|------|----------|---------|
| 22:30 JST | #buildinpublic で10投稿にリプライ | 15分 |
| 23:00 JST | メイン投稿 | 5分 |
| 23:00-23:30 JST | 投稿後、来たリプライに即返信 | 30分 |
| 翌朝 7:00 | 日本語版投稿 + 残りリプライ対応 | 10分 |
| 日中 x2-3回 | マイクロ投稿 | 各2分 |

### リプライのルール
- 投稿前に15分、投稿後30分はリプライに集中
- リプライは「他人のTLに表示される」唯一の方法
- 質問 or 自分の経験を添える（「いいですね」だけは避ける）
- 1日10リプライ最低ライン

### 引用RTの使い方
- 他のbuilderの成果を称える（Give first）
- 自分の意見を1-2行添える
- 週2-3回

## 競合との差別化メッセージ

### stickK / Beeminder / COMMITの違い

| | stickK | Beeminder | COMMIT |
|---|--------|-----------|--------|
| ターゲット | 汎用（ダイエット等） | ナード向け | 読書に特化 |
| UI | 古い（2010年代） | グラフ重視 | Titan（luxury dark） |
| 失敗時 | 慈善団体 or Anti-charity | 自社に支払い | Room to Read（児童教育） |
| 体験 | フォーム入力 | データダッシュボード | Monk Mode、Hall of Fame |
| モバイル | Web only | Web + API | ネイティブアプリ（iOS/Android） |
| 読書特化 | No | No | Yes（ISBN検索、ページ追跡） |

### COMMITのワンライナー
> 読書に特化した、モバイルネイティブのcommitment device。
> 失敗したお金は児童教育に届く。

### 啓蒙メッセージ（99%が「Commitment Device」を知らない）
- 「Commitment Device = 逃げ道を事前に消す仕組み」
- 「stickK（イェール大発）で72%が目標達成」
- 「お金を賭ける → 損失回避バイアスが働く → 行動する」
- 「読書版のstickK、モバイルネイティブ、寄付は児童教育」

## マイクロ投稿テンプレート集

### Type 1: 数字系
```
積読の平均冊数: 17冊。平均放置期間: 2.3年。
```
```
Average tsundoku: 17 books. Average neglect period: 2.3 years.
```

### Type 2: 一行の気づき
```
本を買うのは簡単。読むのが難しいんじゃなくて、始めるのが難しい。
```
```
Buying books is easy. Reading isn't hard — starting is.
```

### Type 3: 開発の一コマ
```
今日のコミット: 47行追加、200行削除。引き算の開発。
```
```
Today's commit: +47 lines, -200 lines. Building by subtraction.
```

### Type 4: 引用RT用コメント
```
これ。読書の問題は意志力じゃなくて、構造の問題。
```
```
This. The reading problem isn't willpower — it's structure.
```

### Type 5: 問いかけ
```
最後に本を読み終えたの、いつだろう。
```
```
When was the last time you actually finished a book?
```

## アルゴリズム対策

| ルール | 理由 |
|--------|------|
| 外部リンクを本文に入れない | インプレッション50%減 |
| 投稿後30分以内にリプライを稼ぐ | 初速がリーチを決める |
| 画像/動画はネイティブアップロード | 外部URLは優先度下がる |
| ハッシュタグは1-2個まで | 多すぎるとスパム判定 |
| リプライ > 自分の投稿 | 他者TLに露出する |
| Community投稿優先 | 0フォロワーでもリーチ可能 |
| 動画は2倍のリーチ | テキストのみより優先表示 |
